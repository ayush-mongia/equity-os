# EquityOS

A portfolio dashboard for Indian investors holding US equities. It shows what a
position is actually worth after India capital-gains tax, INR depreciation, and
inflation — not just the raw USD price move — and includes an AI-assisted
"Slump Verdict" that checks whether a drawdown reflects a real business
problem or is just a tax/currency/inflation artifact.

## Features

- **Real-return engine** — converts raw USD gains into post-tax, currency- and
  inflation-adjusted INR returns (STCG/LTCG classification, surcharge slabs,
  historical vs. current USD/INR).
- **Portfolio dashboard** — holdings table, summary cards, return chart.
- **Slump Verdict** — for a position with a genuine real-return drawdown, the
  app searches the web (via [Tavily](https://tavily.com)) for dated evidence
  and asks Gemini to steelman the bear case from that evidence only, citing
  sources it can actually be checked against — never taking the model's word
  for a citation. Bring-your-own-key for both providers; see below.

## Slump Verdict API keys

Slump Verdict is bring-your-own-key and needs no server-side environment
variables. Open **AI Settings** in the app and add:

- A free [Gemini API key](https://aistudio.google.com/app/apikey)
- A free [Tavily API key](https://app.tavily.com/home) (1,000 searches/month)

Both keys are stored only in your browser (`localStorage` by default, or
`sessionStorage`-only for the session if you enable "Don't remember my keys on
this device" in AI Settings) and are sent only to this app's own server when
you request a verdict, which forwards them to Google/Tavily. They are never
logged or persisted server-side.

## Deployment

Configured for [Render](https://render.com) via `render.yaml` (plain Node
server, not a serverless function). `next.config.ts` sets CSP and other
security headers, and both API routes are rate-limited per IP — see that file
and `src/lib/rateLimit.ts` for details.

### Deploying by uploading files manually (no `git push`)

If you're pushing this to GitHub via the web UI instead of `git`, upload
everything **except** what's in `.gitignore` — mainly:

- `node_modules/` — reinstalled from `package-lock.json` on any machine/CI,
  never commit it (tens of thousands of files).
- `.next/` — build output, regenerated by `npm run build`.
- `*.tsbuildinfo`, `next-env.d.ts` — build/editor-generated.
- `.claude/`, `.vscode/`, `.idea/` — local tool settings, not project code.
- `.DS_Store`, `.env*` — machine junk / secrets. There's no `.env` file in
  this project (all keys are BYOK, entered in-browser), but if you ever add
  one, it must never be committed.

Everything under `src/`, `public/`, plus `package.json`, `package-lock.json`,
`next.config.ts`, `tsconfig.json`, `postcss.config.mjs`, `eslint.config.mjs`,
`.gitignore`, `render.yaml`, `AGENTS.md`, `CLAUDE.md`, and this `README.md`
should go up.

`slump-verdict-spec.pdf` (the feature's design spec) is safe and useful to
include but not required. `India_USA_Tax_Reference (1).xlsx` is a personal
reference file, not part of the app — leave it out unless you specifically
want it in the repo.

## Project structure

```
src/
  app/                  Routes (App Router) + the two API routes
    api/slump-verdict/  Orchestrates Tavily search -> Gemini verdict
    api/search-ticker/  Local ticker autocomplete
  components/           Dashboard UI
  lib/
    tax.ts               India capital-gains tax rules
    calculations.ts       Real-return math (tax + currency + inflation)
    gemini.ts, tavily.ts   AI provider clients
    slumpVerdict.ts        Gate logic, caching, copy for each result state
    storage.ts             Client-side persistence (localStorage/sessionStorage)
    rateLimit.ts            Per-IP rate limiting for the API routes
```

## Note on this Next.js version

This project pins a Next.js version with breaking changes from what you may
expect (e.g. `middleware.ts` is now `proxy.ts`). Check
`node_modules/next/dist/docs/` for the version actually installed before
assuming any API works the way it used to.
